## 优化提示词：基于健康指标表设计后端接口与 Flutter 首页展示

### 背景
你已根据健康指标管理平台的需求，在 PostgreSQL 数据库中创建了以下表：
- `health_indicator_def`：指标定义（代码、名称、单位、正常范围、预警阈值）
- `health_record`：通用指标记录（支持数值、文本、JSONB）
- `blood_pressure_record`、`glucose_record`、`heart_rate_record`、`weight_record`、`spo2_record`：高频专用指标表
- `step_record`、`sleep_record`：活动与睡眠表
- `medication_plan`、`medication_record`：用药计划与执行记录
- `health_alert`：健康预警记录
- `user_device`：用户设备绑定

所有表均已配置 `created_at` / `updated_at` 触发器。

### 目标
1. **后端接口设计**：基于 Spring Boot + MyBatis-Plus（或你使用的后端框架），为上述健康指标表提供 RESTful API，支持：
   - 保存/更新指标数据（手动录入或设备同步）
   - 查询当前用户的最新指标值（用于首页概览）
   - 查询某指标的历史趋势数据（用于图表）
   - 查询异常预警列表
   - 查询用药计划及今日执行情况
   - 查询步数、睡眠等每日汇总数据
   - 设备绑定与管理（可选）

2. **Flutter 首页展示**：设计一个现代、简洁的健康看板，包含：
   - 顶部用户信息（头像、昵称、欢迎语）
   - 关键指标卡片：血压、血糖、心率、体重、血氧、步数、睡眠等（根据用户实际数据动态显示）
   - 每个卡片显示最新值、单位、正常状态（正常/偏高/偏低）及趋势箭头（与上次对比）
   - 异常预警条：滚动显示最新的预警消息，点击可查看详情
   - 用药提醒：显示今日待服用药列表，支持点击打卡
   - 底部导航栏（复用已有的 `BottomNavBar`），当前页为“首页”

### 技术要求
#### 后端（Java/Spring Boot）
- 使用 MyBatis-Plus 作为 ORM 框架
- 使用 `Long` 作为用户 ID（通过 `BaseContext.getCurrentId()` 获取当前登录用户）
- 使用 `LocalDateTime` 处理时间
- 统一返回 `Result<T>` 格式（code, message, data）
- 接口示例：
  - `GET /api/health/latest` → 返回当前用户所有最新指标
  - `GET /api/health/trend?indicatorCode=bp_systolic&days=7` → 返回最近7天血压收缩压趋势数据
  - `POST /api/health/record` → 保存指标记录（支持通用表和专用表）
  - `GET /api/health/alerts` → 返回未处理的预警
  - `GET /api/medication/today` → 返回今日用药计划及执行状态
  - `POST /api/medication/record` → 记录用药打卡

#### 前端（Flutter）
- 使用 `Provider` 或 `Riverpod` 进行状态管理（你已有的 `AuthStore` 等）
- 使用 `lucide_icons` 图标库
- 使用 `fl_chart` 或 `syncfusion_flutter_charts` 展示趋势折线图（可选，首页可先不展示图表，仅展示最新值卡片）
- 支持亮色/暗色主题（使用已有的 `AppTheme`）
- 使用 `RefreshIndicator` 支持下拉刷新
- 卡片采用圆角、阴影，风格类似 iOS 健康 App

### 详细需求
#### 后端接口详细说明
1. **获取最新指标概览**
   - 路径：`GET /health/dashboard`
   - 响应示例：
   ```json
   {
     "code": 200,
     "data": {
       "bp": { "systolic": 128, "diastolic": 82, "pulse": 72, "recordTime": "2025-04-03 08:30:00", "status": "normal" },
       "glucose": { "value": 6.1, "type": "fasting", "recordTime": "2025-04-03 07:00:00", "status": "normal" },
       "heartRate": { "value": 75, "recordTime": "2025-04-03 08:30:00", "status": "normal" },
       "weight": { "value": 68.5, "bmi": 23.4, "recordTime": "2025-04-02 08:00:00", "status": "normal" },
       "steps": { "count": 5200, "goal": 8000, "date": "2025-04-02" },
       "sleep": { "duration": 7.2, "quality": 4, "date": "2025-04-02" }
     }
   }
   ```

2. **保存指标记录**
   - 路径：`POST /health/record`
   - 请求体根据指标类型灵活设计，例如：
   ```json
   {
     "indicatorCode": "bp_systolic",
     "value": 128,
     "recordTime": "2025-04-03 08:30:00",
     "source": "manual",
     "deviceId": null
   }
   ```
   后端根据 `indicatorCode` 自动路由到对应的专用表或通用表。

3. **获取趋势数据**
   - 路径：`GET /health/trend?indicatorCode=bp_systolic&days=7`
   - 返回时间序列数据，用于前端绘制折线图。

4. **异常预警**
   - 路径：`GET /health/alerts?status=0`（未处理）
   - 返回列表，每条包含预警内容、严重程度、指标名称等。

5. **用药计划**
   - 路径：`GET /medication/today`
   - 返回今日需服用的药品列表，以及每个药品的服用时间点、是否已打卡。
